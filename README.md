# Task Scheduler

Многопоточный планировщик задач с автоматической параллелизацией на основе графа зависимостей (DAG).

## 📋 Описание

Task Scheduler — это C++ библиотека для выполнения задач с зависимостями в оптимальном параллельном порядке. Система автоматически определяет, какие задачи могут выполняться одновременно, и распределяет их по пулу рабочих потоков.

### Основные возможности

- ✅ **Автоматическая параллелизация** — задачи без зависимостей выполняются одновременно
- ✅ **Граф зависимостей (DAG)** — определение порядка выполнения на основе зависимостей
- ✅ **Обнаружение циклов** — валидация графа перед выполнением
- ✅ **Thread Pool** — эффективное использование ресурсов CPU
- ✅ **Обработка ошибок** — graceful handling при падении задачи
- ✅ **Статистика выполнения** — детальная информация о времени и статусе каждой задачи
- ✅ **Современный C++17** — использование умных указателей, лямбд, STL

## 💡 Пример использования

```cpp
#include "scheduler.hpp"

int main() {
    // Создать планировщик с 4 рабочими потоками
    Scheduler scheduler(4);
    
    // Добавить независимые задачи (выполнятся параллельно)
    scheduler.add_task("task_A", []() {
        std::cout << "Executing task A" << std::endl;
    });
    
    scheduler.add_task("task_B", []() {
        std::cout << "Executing task B" << std::endl;
    });
    
    // Добавить задачу с зависимостями (выполнится после A и B)
    scheduler.add_task("task_C", {"task_A", "task_B"}, []() {
        std::cout << "Executing task C (depends on A and B)" << std::endl;
    });
    
    // Выполнить все задачи
    scheduler.execute();
    
    // Показать статистику
    std::cout << scheduler.get_statistics() << std::endl;
    
    return 0;
}
```

## 🏗️ Архитектура

### Основные компоненты

1. **Task** (`task.hpp/cpp`)
   - Представляет одну задачу с функцией, зависимостями и статусом
   - Отслеживает время выполнения и ошибки

2. **DAG** (`dag.hpp/cpp`)
   - Граф зависимостей задач
   - Топологическая сортировка
   - Обнаружение циклических зависимостей
   - Определение готовых к выполнению задач

3. **ThreadPool** (`thread_pool.hpp/cpp`)
   - Пул рабочих потоков
   - Очередь задач с FIFO
   - Синхронизация через condition variables

4. **Scheduler** (`scheduler.hpp/cpp`)
   - Высокоуровневый интерфейс
   - Координация выполнения задач
   - Сбор статистики

### Граф зависимостей

```
     A       B       (независимые, выполняются параллельно)
      \     /
       \   /
         C           (зависит от A и B)
         |
         D           (зависит от C)
```

## 🔍 Особенности реализации

### Многопоточность

- Использование `std::thread`, `std::mutex`, `std::condition_variable`
- Lock-free подсчёт активных задач через `std::atomic`
- Избежание deadlock'ов через правильную блокировку

### Обработка ошибок

- Все исключения в задачах перехватываются
- Статус задачи меняется на `FAILED`
- Выполнение останавливается при первой ошибке
- Сообщения об ошибках сохраняются

### Валидация графа

- Проверка существования всех зависимостей
- Обнаружение циклов через DFS
- Выброс исключения при невалидном графе
